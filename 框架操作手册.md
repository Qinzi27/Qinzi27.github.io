# Qinzi27 框架操作手册

这份手册放在项目根目录，用来回答最常见的操作问题：怎么打开预览、去哪里编辑内容、怎么生成小日历、怎么检查和发布。

本项目是一个基于 Quartz v5 的 Markdown 静态网站。日常内容主要写在 `content/` 里；Quartz 框架代码在 `quartz/` 里；生成出来的网站文件在 `public/`、`out/`、`.next/` 等目录里，通常不要手动改这些生成目录。

## 1. 先进入项目目录

👉 需要你人工操作：

打开 PowerShell，进入项目根目录：

```powershell
cd C:\Users\admin\Documents\Qinzi27
```

确认当前位置：

```powershell
Get-Location
```

如果显示的是：

```text
C:\Users\admin\Documents\Qinzi27
```

说明已经在正确位置。

## 2. 第一次使用：安装依赖

👉 需要你人工操作：

如果是第一次在这台电脑上运行，或者 `node_modules/` 不见了，先安装依赖：

```powershell
npm.cmd ci
```

平时已经安装过依赖，就不需要每次都运行。

## 3. 打开本地预览

👉 需要你人工操作：

普通预览：

```powershell
npm.cmd run preview
```

这个命令会依次做几件事：

1. 重新生成小日历页面。
2. 重新生成贴纸墙页面。
3. 运行隐私和发布检查。
4. 安装 Quartz 插件。
5. 启动本地预览服务器。

默认预览端口在 `scripts/preview-site.mjs` 里设置为 `8090`，所以通常打开：

```text
http://localhost:8090
```

如果终端打印了不同地址，以终端显示为准。

关闭预览时，回到运行预览的 PowerShell 窗口，按：

```text
Ctrl + C
```

## 4. 预览受密码保护的小日历

👉 需要你人工操作：

如果要预览 `content/Our Calendar/index.md` 这种 `privacy: "protected"` 的页面，需要先设置环境变量：

```powershell
$env:PARENT_CALENDAR_PASSWORD = "这里换成你的本地预览密码"
npm.cmd run preview
```

注意：不要把真实密码写进 Markdown 文件。受保护页面应使用：

```yaml
passwordEnv: "PARENT_CALENDAR_PASSWORD"
```

## 5. 编辑普通页面内容

普通页面主要在 `content/` 目录里编辑。

常用位置：

```text
content/index.md                         首页
content/About.md                         About 页面
content/Now.md                           Now 页面
content/Learning Notes/index.md          学习笔记栏目首页
content/Research & Papers/index.md       论文阅读栏目首页
content/Life Journal/index.md            生活记录栏目首页
content/Projects/index.md                项目栏目首页
```

每个 Markdown 文件一般分两部分。

第一部分是页面信息，叫 frontmatter：

```yaml
---
title: "页面标题"
date: "2026-06-29"
type: "page"
tags: ["tag1", "tag2"]
status: "active"
publish: true
privacy: "public"
summary: "页面摘要。"
---
```

第二个 `---` 后面才是正文：

```md
# 页面标题

这里开始写真正显示在页面上的内容。
```

发布规则：

```yaml
publish: true
privacy: "public"
```

草稿建议先写成：

```yaml
publish: false
privacy: "private"
```

## 6. 新增一篇普通笔记

👉 需要你人工操作：

推荐从 `templates/` 复制一个模板到 `content/` 下面对应栏目。

常用模板：

```text
templates/learning-note.md
templates/paper-reading-note.md
templates/project-log.md
templates/weekly-review.md
templates/life-journal.md
```

例如新增学习笔记，可以放在：

```text
content/Learning Notes/你的笔记名.md
```

新文件建议先用：

```yaml
publish: false
privacy: "private"
```

等确认没有隐私信息、结构也满意后，再改成公开。

## 7. 编辑小日历每日记录

小日历有两个文件，分工不同：

```text
content/Our Calendar/每日记录编辑本.md    日常编辑入口
content/Our Calendar/index.md            自动生成的受保护日历页面
```

日常应该改：

```text
content/Our Calendar/每日记录编辑本.md
```

不要直接改：

```text
content/Our Calendar/index.md
```

因为它会被生成脚本覆盖。

每日记录格式示例：

```md
## 2026-06-29
sleep: 01:00

### 碎碎念
- 今天一点睡着。
- 会一直爱你。
```

改完后运行：

```powershell
npm.cmd run generate-calendar
```

这个命令会把编辑本同步生成到：

```text
content/Our Calendar/index.md
```

## 8. 使用每日记录 GUI

👉 需要你人工操作：

如果不想手改 Markdown，可以打开 GUI：

```powershell
npm.cmd run daily-gui
```

终端会打印一个本地地址，例如：

```text
Daily log GUI: http://127.0.0.1:5177
```

打开终端显示的地址。端口可能变化，以终端输出为准。

GUI 有两个常用页面：

```text
/                快速新增记录
/calendar.html   查看、搜索、编辑已有日期
```

保存时它会写入：

```text
content/Our Calendar/每日记录编辑本.md
```

并自动备份到：

```text
content/private/backups/daily-log/
```

关闭 GUI 时，回到运行 GUI 的 PowerShell 窗口，按：

```text
Ctrl + C
```

## 9. 生成贴纸墙

贴纸资源主要在：

```text
content/assets/stickers/
content/assets/couple-calendar-stickers/
```

手动重新生成贴纸墙：

```powershell
npm.cmd run generate-sticker-wall
```

手动重新生成小日历贴纸和日历：

```powershell
npm.cmd run generate-calendar
```

平时运行 `npm.cmd run preview` 或 `npm.cmd run build` 时，也会自动执行这些生成步骤。

## 10. 检查项目

👉 需要你人工操作：

发布前建议运行：

```powershell
npm.cmd run check
```

它会执行：

```text
npm run prepublish-check
tsc --noEmit
```

含义：

- `prepublish-check` 检查可发布内容里是否有敏感词、错误密码字段、被排除的文件类型等。
- `tsc --noEmit` 做 TypeScript 类型检查，不生成新文件。

如果只想跑隐私和发布检查：

```powershell
npm.cmd run prepublish-check
```

## 11. 构建正式网站

👉 需要你人工操作：

构建静态网站：

```powershell
npm.cmd run build
```

这个命令会：

1. 生成小日历。
2. 生成贴纸墙。
3. 运行发布前检查。
4. 安装 Quartz 插件。
5. 构建最终静态网站。

构建结果主要写入：

```text
public/
```

一般不要手动编辑 `public/`，因为下次构建会覆盖。

## 12. 查看 Git 改动

👉 需要你人工操作：

提交前先看有哪些文件变了：

```powershell
git status
```

查看某个文件具体改了什么：

```powershell
git diff -- content/index.md
```

如果只是想看全部改动：

```powershell
git diff
```

## 13. 提交和推送

👉 需要你人工操作：

确认改动无误后：

```powershell
git add .
git commit -m "Update site content"
git push
```

如果只想提交某几个文件，建议精确添加：

```powershell
git add content/index.md
git add 框架操作手册.md
git commit -m "Add operation manual"
git push
```

注意：`git add .` 会加入当前项目里的所有改动。提交前一定先看 `git status`。

## 14. 常见问题

### 预览打不开

先看终端有没有打印本地地址。默认一般是：

```text
http://localhost:8090
```

如果端口被占用，可以临时换端口：

```powershell
$env:QUARTZ_PREVIEW_PORT = "8091"
npm.cmd run preview
```

然后打开：

```text
http://localhost:8091
```

### 改了内容但页面没变化

优先检查：

1. 是否改的是 `content/` 里的源文件。
2. 是否保存了文件。
3. 小日历是否运行了 `npm.cmd run generate-calendar`。
4. 浏览器是否需要刷新。
5. 终端里的预览服务是否还在运行。

### 小日历改了又被覆盖

通常是因为直接改了：

```text
content/Our Calendar/index.md
```

正确入口是：

```text
content/Our Calendar/每日记录编辑本.md
```

### 发布检查失败

先看终端报错里给出的文件名和行号。常见原因：

- 公开页面里出现 `password`、`secret`、`token`、`API key` 等敏感词。
- 受保护页面没有设置 `passwordEnv`。
- Markdown 里写了真实密码。
- 把不该发布的附件放进了 `content/` 的公开区域。

## 16. 推荐日常流程

普通内容更新：

```text
编辑 content/ 里的 Markdown
运行 npm.cmd run preview
浏览器检查页面
运行 npm.cmd run check
git status
提交并推送
```

小日历更新：

```text
编辑 content/Our Calendar/每日记录编辑本.md
运行 npm.cmd run generate-calendar
运行 npm.cmd run preview
浏览器检查小日历
git status
提交并推送
```

GUI 小日历更新：

```text
运行 npm.cmd run daily-gui
打开终端给出的本地地址
新增或修改日期记录
保存并同步日历页面
运行 npm.cmd run preview 检查
git status
提交并推送
```
